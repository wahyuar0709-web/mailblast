💡 Panduan: Email Lamaran yang Dilirik HRD
📌 Subject jelas & spesifik — HRD menyaring ratusan email lewat subject. Format terbaik: Lamaran Kerja - [Posisi] - [Nama Kamu]
📌 Body singkat, maks 150–200 kata — HRD scan email dalam 5–10 detik. Paragraf panjang berisiko di-skip.
📌 Sebutkan posisi & sumber info loker — Tunjukkan kamu melamar posisi spesifik, bukan email massal generik.
📌 Cantumkan 1–2 poin kekuatan utama — Skill/pengalaman paling relevan, bukan riwayat hidup lengkap (itu fungsi CV).
📌 Sertakan kontak aktif — No. HP & email yang sering dicek, supaya gampang dihubungi untuk tes.
📌 CTA jelas di akhir — "Saya siap mengikuti tahap seleksi/wawancara" lebih kuat daripada sekadar "terima kasih".
📌 Lampiran lengkap & rapi — CV + portofolio (jika ada). File bernama jelas: CV_NamaKamu.pdf, bukan scan001.pdf.
⚠️ Hindari: huruf kapital berlebihan, banyak tanda seru, font warna-warni, dan badan email yang sama persis untuk semua PT tanpa personalisasi posisi.
🔖 Placeholder Otomatis — Cara Kerja & Cara Isi

Placeholder ini otomatis terisi berbeda untuk tiap PT saat email dikirim. Kamu tidak perlu menulis ulang body untuk tiap lowongan — cukup isi data per-PT di halaman Perusahaan.

{NAMA_PT}
Diambil dari kolom "Nama PT" saat kamu menambah perusahaan di halaman Perusahaan.
{POSISI}
Diambil dari kolom "Posisi" per-PT. Ini kunci untuk lowongan yang beda-beda — isi posisi sesuai info loker yang kamu lihat (LinkedIn/Jobstreet/dll) saat menambah PT itu.
{EMAIL}
Email tujuan (HRD) PT tersebut, otomatis dari data PT.
💬 Lowongan beda posisi tiap PT? Tidak perlu bikin template baru. Cukup pastikan kolom Posisi diisi benar saat tambah/edit PT — body email ini otomatis menyesuaikan lewat {POSISI}. Untuk PT dengan kebutuhan body yang benar-benar berbeda (bukan cuma posisi), pakai "Body Email Khusus" di form edit PT tersebut.

📅 Jadwal Pengiriman Email Lamaran ⚠️ Tips Anti-Spam
⚠️ Tips Anti-Spam & Anti-Blokir Gmail
• Jangan kirim setiap hari — idealnya 2–3 hari per minggu dengan jeda antar hari
• Batasi maksimal 30–50 email/hari (Gmail limit: 500/hari, tapi lebih aman lebih sedikit)
• Gunakan delay antar email min. 30–60 detik agar tidak terdeteksi bot
• Variasikan isi email dengan fitur A/B template

📬 Deteksi Balasan Email HRD
Sistem memeriksa inbox Gmail untuk balasan dari perusahaan yang sudah dikirim lamaran. Balasan dideteksi berdasarkan email pengirim. Tandai status atau hapus dari daftar PT langsung di sini.

ℹ️ Syarat: GAS kamu harus sudah memiliki fungsi checkReplies.  Lihat kode GAS
Tambahkan ke Google Apps Script:
function checkReplies(emails) {
  var results = [];
  emails.forEach(function(pt) {
    var threads = GmailApp.search('from:(' + pt.email + ')', 0, 3);
    if (threads.length > 0) {
      var msg = threads[0].getMessages().slice(-1)[0];
      var body = msg.getPlainBody().substring(0, 300);
      var from = msg.getFrom();
      var isBounced = /mailer-daemon|delivery.status|undelivered|no.such.user|invalid.address|does.not.exist|account.*not.*exist/i.test(body + ' ' + from);
      var isInterview = /interview|panggilan|tes|test|jadwal|undangan|hadir/i.test(body);
      results.push({ id: pt.id, email: pt.email, nama: pt.nama,
        hasReply: true, isBounced: isBounced, isInterview: isInterview,
        snippet: body, replyFrom: from,
        replyDate: msg.getDate().toISOString() });
    }
  });
  return results;
}

Selamat datang di MailBlast!

          

            Aplikasi ini membantu kamu kirim lamaran kerja ke ratusan perusahaan secara otomatis.


            Setup hanya butuh 2 menit. Ikuti langkah-langkah berikut.
          

        

        

          
Yang akan kamu setup:

          

            
⬜ Koneksi ke Firebase (database realtime — semua data)

            
⬜ Koneksi ke Google Apps Script (kirim email saja)

            
⬜ Nama & email pengirim

          

        

        
      


      
      

        
🔥 Hubungkan ke Firebase

        

          Firebase menjadi database realtime MailBlast — data langsung sync antar perangkat tanpa refresh.
        


        

          
📋 Cara mendapatkan config:

          
1. Buka console.firebase.google.com

          
2. Buat project baru → aktifkan Realtime Database

          
3. Klik ⚙️ Project Settings → tab General

          
4. Scroll ke "Your apps" → klik ikon Web (</>)

          
5. Copy nilai databaseURL, apiKey, projectId

        


        

          

            
            
          

          

            
            
          

          

            
            
          

          

            
            
          

        


        


        

          
          
          
        

        

          
        

      


      
      

        
Hubungkan ke Google Apps Script

        

          GAS dipakai khusus untuk kirim email via Gmail — tidak bisa digantikan karena hanya GAS yang punya akses MailApp.
        


        

          
📋 Cara mendapatkan URL:

          
1. Buka script.google.com

          
2. Buat project baru → paste kode MailBlast.gs

          
3. Klik Deploy → New deployment

          
4. Pilih type: Web app → Execute as: Me

          
5. Copy URL yang muncul → paste di bawah
